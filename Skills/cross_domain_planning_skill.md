# Cross-Domain Planning Skill

## Purpose
Coordinates planning activities that span multiple domains (Communication, Operations, Accounting/Tracking) in the Gold Tier AI Employee system.

## Inputs
- `primary_domain`: Main domain for the task (communication, operations, accounting)
- `secondary_domains`: Additional domains involved in the task
- `domain_dependencies`: Dependencies between different domains
- `coordination_requirements`: Specific coordination needs between domains
- `approval_points`: Points requiring cross-domain approval
- `resource_conflicts`: Potential conflicts in resource usage

## Outputs
- `coordinated_plan`: Unified plan spanning multiple domains
- `domain_sequence`: Optimal sequence for domain operations
- `resource_schedule`: Allocated resources across domains
- `dependency_map`: Map of inter-domain dependencies
- `approval_workflow`: Cross-domain approval requirements
- `risk_assessment`: Risks associated with cross-domain operations

## Functionality
1. Identifies all domains involved in a multi-domain task
2. Maps dependencies between different domain operations
3. Sequences operations to respect domain constraints
4. Allocates resources across domains to prevent conflicts
5. Defines cross-domain approval points
6. Assesses risks associated with multi-domain operations
7. Creates coordinated execution timeline
8. Establishes communication protocols between domains

## Domain Coordination Process
1. Discovers all involved domains from task requirements
2. Analyzes domain-specific constraints and capabilities
3. Identifies inter-domain dependencies and data flows
4. Plans resource allocation to prevent conflicts
5. Designs approval workflow spanning multiple domains
6. Establishes error handling across domain boundaries
7. Creates monitoring and verification procedures

## Communication Domain Integration
- Email and messaging coordination
- Recipient verification across domains
- Content consistency across channels
- Approval requirements for new recipients

## Operations Domain Integration
- File and project management coordination
- Resource allocation and scheduling
- Cross-system data consistency
- Operational dependency management

## Accounting/Tracking Domain Integration
- Activity logging across domains
- Cost tracking for multi-domain operations
- Performance metrics aggregation
- Compliance reporting coordination

## Integration Points
- Connects with all domain-specific processors
- Integrates with approval workflow for cross-domain actions
- Coordinates with audit system for multi-domain tracking
- Links to resource management for allocation
- Communicates with dashboard for cross-domain status

## Safety Constraints
- Ensures domain isolation where required
- Maintains security boundaries between domains
- Prevents unauthorized cross-domain access
- Enforces approval requirements for cross-domain actions
- Maintains audit trails across domain boundaries
- Respects resource allocation constraints
