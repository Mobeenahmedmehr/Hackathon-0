# AI Employee Usage Guide

## Getting Started

The AI Employee Bronze Tier system operates on a simple file-based workflow. Here's how to use it:

### 1. System Components

- **Inbox/**: Place files here to be detected by the system
- **Needs_Action/**: Task files are created here for processing
- **Done/**: Completed tasks are moved here
- **Plans/**: Strategic plans generated by the system
- **Logs/**: System logs and audit trails
- **Skills/**: Claude Agent Skills definitions
- **Watchers/**: File system monitoring scripts

### 2. Basic Workflow

1. Place a file in the `Inbox/` folder
2. The file watcher detects the new file
3. A task file is created in `Needs_Action/`
4. The task processor handles the task
5. The task is moved to `Done/`
6. Dashboard and logs are updated

### 3. Creating Custom Tasks

To create a custom task for the AI Employee, create a markdown file in the `Inbox/` folder with the following structure:

```markdown
# Task: [Task Title]

## Instructions
Detailed instructions for what the AI should do.

## Expected Outcomes
What success looks like for this task.

## Additional Notes
Any supplementary information.
```

### 4. Running the System

Start the AI Employee system with:

```bash
python run_ai_employee.py
```

The system will continuously monitor for new files and process tasks.

### 5. Monitoring Progress

Check the `Dashboard.md` file for:
- System status
- Pending task count
- Completed task count
- System health indicators

Check the `Logs/` folder for detailed processing information.

### 6. Task Types

The system recognizes different types of tasks:

- **Standard Tasks**: General processing requests
- **Planning Tasks**: Tasks containing "plan", "strategy", or "outline" trigger plan generation
- **File Operations**: Tasks requesting file manipulations within allowed boundaries

### 7. Safety Features

- All actions are logged for audit trail
- System operates only within designated folders
- No external connections or API calls
- File integrity preserved during operations
- Error handling prevents system crashes

### 8. Troubleshooting

If the system doesn't respond to new files:

1. Check that the file watcher is running
2. Verify file permissions for the folder system
3. Review logs in the `Logs/` folder
4. Ensure the file is in the correct format

### 9. Stopping the System

Press `Ctrl+C` to safely stop the AI Employee system. This will terminate both the watcher and processor components.

### 10. Extending Functionality

To extend the system:

1. Add new skills to the `Skills/` folder
2. Modify the task processor for new capabilities
3. Update the dashboard updater for new metrics
4. Enhance the watcher for new trigger conditions
